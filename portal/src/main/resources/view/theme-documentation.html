Grille classique

La classe défini une nouvelle ligne contenant des cellules. Chaque élément doit spécifier une largeur allant de un à douze, en anglais, c'est-à-dire utiliser les classes à .


  <div class="row">
	<div class="two cell">Two cell</div>
  	<div class="ten cell">Ten cell</div>
  </div>

Ajouter une hauteur, allant de à , va définir la hauteur de l'élément en fonction de la largeur de son conteneur. Cela signifie que est un carré.

Le contenu à l'intérieur d'un bloc height-* doit être placée dans une div ayant la classe .

Problèmes connus

Ajouter un padding dans un bloc détruit son positionnement. Le doit être appliqué à la div qu'il contient, ou à un wrapper. Vous pouvez aussi travailler avec la propriété .

Cet exemple ne marche pas : 


  <div class="row">
	<div class="twelve cell height-two" style="padding:15px">
	  <div class="absolute cell">Twelve cell height-two avec un padding de 15px</div>
	</div>
  </div>
Twelve cell height-two avec un padding de 15px

Cet exemple marche : 


  <div class="row">
	<div class="twelve cell height-two">
	  <div class="absolute cell" style="padding:15px">
		Twelve cell height-two, avec un padding de 15px padding dans le bloc absolute
	  </div>
	</div>
  </div>
Twelve cell height-two, avec un padding de 15px padding dans le bloc absolute

Cet exemple marche aussi : 


  <div class="row">
    <div class="twelve cell" style="padding:15px;background:#333;">
      <div class="twelve cell height-two">
        <div class="absolute cell">Twelve cell height-two, avec un padding de 15px padding dans un wrapper</div>
	  </div>
    </div>
  </div>

LESS usage

LESS files are included into each other so that the app.css from the theme contains all CSS.

In the public structure, lib contains grid classes as well as various mixins meant to be used as functions. Modules specifies the positioning of CSS components (navigation, lists...). App contains the overall structure and serves as a main file meant to be delivered alone.

In the theme structure, elements styles all individual elements (h1, a, ...) and defines colours specifications. Modules is the file defining the styling of CSS components (it's the colours counterpart of modules in structure). Theme is the main file and can be included in a view on its own.

Icons

Icons can be defined via a mixin. Path is optional and defaults to theme/img/icons. Size is applied for both height and width, as icons are assumed to be squared.

	.myicon{
		.icon(@myIconFileName, @size, @pathToIconsFolder);
	}
