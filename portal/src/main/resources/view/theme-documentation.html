LESS usage

LESS files are included into each other so that the app.css from the theme contains all CSS.

In the public structure, lib contains grid classes as well as various mixins meant to be used as functions. Modules specifies the positioning of CSS components (navigation, lists...). App contains the overall structure and serves as a main file meant to be delivered alone.

In the theme structure, elements styles all individual elements (h1, a, ...) and defines colours specifications. Modules is the file defining the styling of CSS components (it's the colours counterpart of modules in structure). Theme is the main file and can be included in a view on its own.

Icons

Icons can be defined via a mixin. Path is optional and defaults to theme/img/icons. Size is applied for both height and width, as icons are assumed to be squared.

	.myicon{
		.icon(@myIconFileName, @size, @pathToIconsFolder);
	}

Prefixes shorthands

Those mixins apply their corresponding CSS3 properties, with all vendor prefixes.

	.run-transition(@property, @speed);
	.rounded-corners(@radius);
	.rounded-corners-left(@radius);
	.rounded-corners-right(@radius);
	.rounded-corners-top(@radius);
	.rounded-corners-bottom(@radius);
	.vertical-gradient(@topColor, @bottomColor);
